Ejercicio Prctica 7: curvas de aprendizaje en Machine Learning: the California housing problem

Una curva de aprendizaje en Machine Learning simplemente es una grfica donde se representa las epochs (eje horizontal) frente a la funcin objetivo (eje vertical).

En este ejercicio vermos cmo usar las curvas de aprendizaje para analizar el comportamiento de varios algoritmos de optimizacin. Para ello, estudiaremos el llamado California housing problem, el cual es un problema de regresin que consiste en predecir el valor que debe tener una casa en California dependiendo de varias caractersticas. La descripcin de los datos es la siguiente:

Labels

'MedHouseVal' = Mediana de viviendas para familias en una manzana (medido en dlares estadounidenses)

Features

'MedInc' = Media de ingresos para grupos familiares en una manzana (medido en decenas de miles de dlares estadounidenses)

'HouseAge' = Antigedad media de una casa

'AveRooms' = Media de habitaciones en un bloque de casas

'AveBedrms' = Media de habitaciones en un barrio

'Population' = Cantidad total de personas que residen en un barrio

'AveOccup' = media de casa ocupadas

'Latitude' = latitud geogrfica de la casa

'Longitude' = longitud geogrfica de la casa

Estas dos ltimas caractersticas hacen referencia a la localizacin de la casa

Las siguientes lneas de cdigo hacen la separacin entre datos de entrenamiento, validacin y test.

Tambin normalizan los datos. Esta es una tcnica usual en Machine Learning que trata de evitar sesgos en los datos de partida. Para cada una de las 8 columnas de caractersticas, la normalizacin de los datos se hace de manera stndard del siguiente modo:

  1. Se calculan la media $\mu_j$ y la desviacin tpica $\sigma_j$, $1\leq j\leq 8$.

  2. Si $x^j$ denota la columna $j-$sima, entonces dicha columna se transforma en

$$ \hat{x}^j = \frac{x^j - \mu_j}{\sigma_j} $$

con lo cual los nuevos datos tienen media cero y varianza 1.

A continuacin, construimos el modelo de prediccin (la red neuronal). Se trata de un MultiLayerPerceptron (MLP) con $8$ canales de entrada, una sla capa oculta de $30$ neuronas, y una salida escalar. Es decir $$ NN(x,\theta) = \sum_{j=1}^{30} a_j\sigma \left(\omega x + b\right)_j + b_{output}, \quad \theta = (\omega; b) $$ donde $\sigma$ es la funcin de activacin, la cual se aplica componente a componente, $\omega$ es una matriz de tamao $30\times 8$, $b$ es un vector bias de tamao $30$, y $b_{output}$ es un bias de salida escalar. Por tanto, nuestro modelo tiene $30\times 8 + 30 + 30 +1 = 301 $ parmetros de entrenamiento.

En el cdigo, el mtodo fit() de Keras usa mini-batch por defecto, incluso si no se especifica explcitamente. La cantidad de datos que contiene cada mini-batch est controlada por el parmetro batch_size, que no se ha definido en este caso. Si no se especifica, Keras utiliza un valor predeterminado batch_size=32.

Por lo tanto, s, se ha utilizado mini-batch en el gradiente estocstico, y cada mini-batch contiene 32 datos.
